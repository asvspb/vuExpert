# Директория styles

Директория `styles` содержит файлы стилей, написанные на SCSS, которые используются в приложении.

## Описание файлов

- `main.scss` - основной файл стилей, который может содержать импорты других стилевых файлов и глобальные стили
- `mixins.scss` - файл с миксинами SCSS, которые можно использовать для повторного использования стилевых фрагментов
- `variables.scss` - файл с переменными SCSS, содержащий цвета, размеры и другие параметры для стилизации

## Назначение

Эта директория отвечает за стилизацию приложения. Использование SCSS позволяет создавать более гибкие и поддерживаемые стили с возможностью использования переменных, миксинов и других функций препроцессора.

## BEM и линтинг стилей

В проекте включён Stylelint с поддержкой Vue SFC и SCSS, а также правилами для BEM-нейминга:
- Блок, элемент, модификатор: `block`, `block__element`, `block--modifier`, `block__element--modifier`
- Допускаются утилиты с префиксом `u-` и состояния с префиксом `is-`
- Допустимы текущие утилиты проекта (allowlist) — планируется миграция к `u-*`

Линтинг:
- npm run lint — ESLint (включая `eslint-plugin-vuejs-accessibility`) и Stylelint
- npm run lint:css — только Stylelint
- npm run lint:css:fix — авто-исправление форматирования и некоторых ошибок

Рекомендации:
- Один корневой блок на компонент (`.component-name`)
- Элементы и модификаторы через `__` и `--`
- Состояния через `is-*`
- Обычно до 3–4 уровней вложенности SCSS

Советы по использованию в Vue-компонентах:
- Один корневой блок на компонент (`.component-name`)
- Элементы и модификаторы оформлять через `__` и `--`
- Состояния при необходимости через `is-*`
- Не более 2–3 уровней вложенности SCSS
